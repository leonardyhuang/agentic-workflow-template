# Runbooks

Step-by-step operational guides for this project.

Add runbooks for:
- Common debugging procedures
- Deployment steps
- Adding new resource types / features
- Incident response playbooks
- Environment setup / onboarding

Format: one markdown file per runbook, named descriptively (e.g., `debug-data-corruption.md`, `deploy-production.md`, `add-new-entity.md`).

Each runbook should include:
1. **When to use this** — trigger conditions
2. **Prerequisites** — access, tools, knowledge
3. **Steps** — numbered, copy-paste friendly where possible
4. **Verification** — how to confirm it worked
5. **Escalation** — when to stop and ask for help
